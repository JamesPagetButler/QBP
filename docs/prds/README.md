# Product Requirements Documents (PRDs)

This directory contains PRDs generated by the Gemini review process for feature implementations.

## Overview

PRDs are generated when Gemini reviews a PR that introduces:
- New experiments
- New visualization components
- New API surfaces
- Significant new features

## Workflow

```
Gemini Review Complete
        ↓
PRD Generated (if applicable)
        ↓
PRD stored in docs/prds/
        ↓
Issues generated from PRD
        ↓
Steve executes Issues
        ↓
Red Team reviews Steve's work
        ↓
Human merge authorization
```

---

## PRD Template

See `gemini_review_prompt.md` for the full PRD template.

### Required Sections

1. **Problem Statement** — What problem does this solve?
2. **Success Criteria** — High-level goals
3. **Technical Approach** — Design from Furey/Feynman perspective
4. **Acceptance Criteria** — Specific, testable requirements
5. **Visualization Requirements** — Video tiers, interactive needs
6. **Issues to Generate** — Action items for implementation
7. **Dependencies** — Required libraries or prior work
8. **Out of Scope** — Explicit boundaries

## Naming Convention

```
prd_NNN_[feature_name].md
```

Examples:
- `prd_001_stern_gerlach_viz.md`
- `prd_002_angle_test_viz.md`
- `prd_003_batch_export.md`

## Status Values

| Status | Meaning |
|--------|---------|
| **Draft** | PRD created, awaiting review |
| **Approved** | PRD approved, ready for implementation |
| **In Progress** | Implementation underway |
| **Complete** | All acceptance criteria met |
| **Superseded** | Replaced by a newer PRD |
